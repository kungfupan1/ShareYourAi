## 一、项目概述

### 1.1 项目背景

ShareYourAi 是一个分布式 AI 内容生成节点调度平台。核心思路：

```
用户需要生成 AI 内容（视频/图片）
    ↓
平台调度空闲节点
    ↓
节点操作 AI 网页完成生成
    ↓
后端校验结果有效性
    ↓
节点获得收益（需审核）
```

#### 2.1.3 插件端链路验证流程

```
┌─────────────────────────────────────────────────────────────┐
│                  插件端链路验证流程                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  核心思路：不截图，改用网络请求监听                          │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  【方案：监听 AI 页面的网络请求】                            │
│                                                             │
│  原理：                                                      │
│  - Grok/Sora 等 AI 在生成视频时，会发起特定的 API 请求      │
│  - 例如：POST /api/generate 返回 task_id                    │
│  - 然后轮询 GET /api/status/{task_id} 获取状态              │
│  - 最终返回视频下载 URL                                     │
│                                                             │
│  插件监听这些请求，记录完整链路：                            │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   请求监听流程                       │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │                                                     │   │
│  │  Step 1: 用户点击生成                               │   │
│  │  ─────────────────                                  │   │
│  │  监听到：POST /api/generate                         │   │
│  │  记录：                                             │   │
│  │    - request_time: 请求时间                         │   │
│  │    - request_body: 请求体（含 prompt）              │   │
│  │    - ai_task_id: AI 返回的任务ID                    │   │
│  │                                                     │   │
│  │  Step 2: AI 处理中                                  │   │
│  │  ─────────────────                                  │   │
│  │  监听到：GET /api/status/{ai_task_id}               │   │
│  │  记录：                                             │   │
│  │    - status_checks: [时间戳列表]                    │   │
│  │    - final_status: completed                        │   │
│  │                                                     │   │
│  │  Step 3: 视频生成完成                               │   │
│  │  ─────────────────                                  │   │
│  │  监听到：响应中包含 video_url                        │   │
│  │  记录：                                             │   │
│  │    - video_url: AI 返回的视频地址                   │   │
│  │    - video_detected_time: 检测到视频的时间          │   │
│  │                                                     │   │
│  │  Step 4: 下载视频                                   │   │
│  │  ─────────────────                                  │   │
│  │  从 AI 的 CDN 下载视频：                            │   │
│  │    - 直接下载 video_url                             │   │
│  │    - 记录 download_time, file_size                  │   │
│  │    - 校验文件头是否为视频格式                        │   │
│  │                                                     │   │
│  │  Step 5: 上传 COS                                   │   │
│  │  ─────────────────                                  │   │
│  │  上传到 COS 指定路径                                │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  最终提交的证据数据：                                        │
│  {                                                          │
│    task_id: "T001",                                         │
│    status: "success",                                       │
│    result_url: "https://cos.../T001.mp4",                  │
│    proof: {                                                 │
│      ai_task_id: "grok_abc123",        // AI返回的任务ID   │
│      request_time: 1234567890,                             │
│      response_time: 1234568090,                            │
│      video_url_original: "https://grok-cdn.../abc.mp4",   │
│      status_checks: [                                      │
│        { time: 1234567900, status: "processing" },        │
│        { time: 1234568000, status: "processing" },        │
│        { time: 1234568090, status: "completed" }          │
│      ],                                                    │
│      download_time: 1234568150,                            │
│      upload_time: 1234568200,                              │
│      total_duration: 310,                                  │
│      video_size: 25600000                                  │
│    }                                                        │
│  }                                                          │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  【Chrome Manifest V3 实现方式】                             │
│                                                             │
│  ⚠️ 重要限制：                                               │
│  在 Manifest V3 中，chrome.webRequest API 无法读取响应体    │
│  （Response Body）！只能知道请求发生了，但拿不到返回数据。   │
│                                                             │
│  ✅ 正确方案：在 content.js 中重写 fetch/XHR                 │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           content.js 请求拦截核心代码                │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │                                                     │   │
│  │  // 保存原始 fetch                                  │   │
│  │  const originalFetch = window.fetch;                │   │
│  │                                                     │   │
│  │  // 重写 fetch，穿上"马甲"                          │   │
│  │  window.fetch = async function(...args) {           │   │
│  │      const [url, options] = args;                   │   │
│  │                                                     │   │
│  │      // 执行原始请求                                │   │
│  │      const response = await originalFetch(...args); │   │
│  │                                                     │   │
│  │      // 克隆响应（因为响应流只能读一次）            │   │
│  │      const clonedResponse = response.clone();       │   │
│  │                                                     │   │
│  │      // 判断是否是我们要监听的 API                  │   │
│  │      const urlStr = url.toString();                 │   │
│  │                                                     │   │
│  │      // 拦截生成请求                                │   │
│  │      if (urlStr.includes('/api/generate')) {        │   │
│  │          const data = await clonedResponse.json();  │   │
│  │          // 上报：AI 任务 ID                        │   │
│  │          window.postMessage({                       │   │
│  │              type: 'AI_REQUEST_CAPTURED',           │   │
│  │              api: 'generate',                       │   │
│  │              data: data,                            │   │
│  │              timestamp: Date.now()                  │   │
│  │          }, '*');                                   │   │
│  │      }                                              │   │
│  │                                                     │   │
│  │      // 拦截状态查询                                │   │
│  │      if (urlStr.includes('/api/status/')) {         │   │
│  │          const data = await clonedResponse.json();  │   │
│  │          window.postMessage({                       │   │
│  │              type: 'AI_STATUS_CAPTURED',            │   │
│  │              api: 'status',                         │   │
│  │              data: data,                            │   │
│  │              timestamp: Date.now()                  │   │
│  │          }, '*');                                   │   │
│  │      }                                              │   │
│  │                                                     │   │
│  │      return response;                               │   │
│  │  };                                                 │   │
│  │                                                     │   │
│  │  // 同样处理 XMLHttpRequest（兼容旧式请求）         │   │
│  │  const originalXHR = window.XMLHttpRequest;         │   │
│  │  window.XMLHttpRequest = function() {               │   │
│  │      const xhr = new originalXHR();                 │   │
│  │      const originalOpen = xhr.open;                 │   │
│  │      xhr.open = function(method, url, ...rest) {    │   │
│  │          xhr._url = url;                            │   │
│  │          return originalOpen.call(xhr, method, url, │   │
│  │              ...rest);                              │   │
│  │      };                                             │   │
│  │      // 拦截响应...（略，类似 fetch 逻辑）          │   │
│  │      return xhr;                                    │   │
│  │  };                                                 │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           消息监听与上报（content.js）               │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │                                                     │   │
│  │  // 收集的证据数据                                  │   │
│  │  const capturedEvidence = {                         │   │
│  │      generateRequest: null,                         │   │
│  │      statusChecks: [],                              │   │
│  │      videoUrl: null                                 │   │
│  │  };                                                 │   │
│  │                                                     │   │
│  │  // 监听拦截到的消息                                │   │
│  │  window.addEventListener('message', (event) => {    │   │
│  │      if (event.source !== window) return;           │   │
│  │                                                     │   │
│  │      const msg = event.data;                        │   │
│  │                                                     │   │
│  │      if (msg.type === 'AI_REQUEST_CAPTURED') {      │   │
│  │          // 记录生成请求                            │   │
│  │          capturedEvidence.generateRequest = {       │   │
│  │              timestamp: msg.timestamp,              │   │
│  │              ai_task_id: msg.data.task_id,          │   │
│  │              request_body: msg.data.prompt          │   │
│  │          };                                         │   │
│  │      }                                              │   │
│  │                                                     │   │
│  │      if (msg.type === 'AI_STATUS_CAPTURED') {       │   │
│  │          // 记录状态查询                            │   │
│  │          capturedEvidence.statusChecks.push({       │   │
│  │              timestamp: msg.timestamp,              │   │
│  │              status: msg.data.status                │   │
│  │          });                                        │   │
│  │                                                     │   │
│  │          // 检测是否完成                            │   │
│  │          if (msg.data.status === 'completed' &&      │   │
│  │              msg.data.video_url) {                  │   │
│  │              capturedEvidence.videoUrl =            │   │
│  │                  msg.data.video_url;                │   │
│  │          }                                          │   │
│  │      }                                              │   │
│  │  });                                                │   │
│  │                                                     │   │
│  │  // 上报证据给 background script                    │   │
│  │  function reportEvidence(taskId) {                  │   │
│  │      chrome.runtime.sendMessage({                   │   │
│  │          action: 'SUBMIT_EVIDENCE',                 │   │
│  │          taskId: taskId,                            │   │
│  │          evidence: capturedEvidence                 │   │
│  │      });                                            │   │
│  │  }                                                  │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  manifest.json 配置：                                       │
│  {                                                          │
│    "manifest_version": 3,                                   │
│    "content_scripts": [                                     │
│      {                                                      │
│        "matches": ["https://grok.com/*"],                   │
│        "js": ["content.js"],                                │
│        "run_at": "document_start"                           │
│      }                                                      │
│    ],                                                       │
│    "permissions": ["storage", "activeTab"],                 │
│    "host_permissions": [                                    │
│      "*://grok.com/*",                                      │
│      "*://*.grok.com/*"                                     │
│    ]                                                        │
│  }                                                          │
│                                                             │
│  注意：run_at: "document_start" 确保 fetch 重写在页面       │
│  加载前就生效，不会漏掉任何请求。                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2.1.4 后端校验流程

```
┌─────────────────────────────────────────────────────────────┐
│                    后端校验流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  收到 success 响应后，执行以下校验：                         │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 1: 时间合理性                                   │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ - total_duration >= 模型最小耗时（如 60 秒）         │   │
│  │ - total_duration <= 模型最大耗时（如 600 秒）        │   │
│  │ - 各步骤时间顺序合理（不能倒流）                     │   │
│  │ - status_checks 数量合理（不是瞬间返回）            │   │
│  └─────────────────────────────────────────────────────┘   │
│                         ↓                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 2: 文件有效性（关键优化！）                     │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │                                                      │   │
│  │  ⚠️ 性能陷阱：                                       │   │
│  │  如果下载整个 20MB 文件只为了读 32 字节文件头，      │   │
│  │  几十个并发就能跑满带宽和内存！                      │   │
│  │                                                      │   │
│  │  ✅ 正确做法：使用 HTTP Range 请求                   │   │
│  │                                                      │   │
│  │  # Python 示例                                       │   │
│  │  import requests                                     │   │
│  │                                                      │   │
│  │  def validate_file_header(url):                      │   │
│  │      # 只请求前 32 字节                              │   │
│  │      response = requests.get(                        │   │
│  │          url,                                        │   │
│  │          headers={'Range': 'bytes=0-31'},            │   │
│  │          timeout=10                                  │   │
│  │      )                                               │   │
│  │      header_bytes = response.content                 │   │
│  │                                                      │   │
│  │      # 校验文件头                                    │   │
│  │      if header_bytes[4:8] == b'ftyp':               │   │
│  │          return 'mp4'                                │   │
```

### 4.8 邮箱验证码技术方案

**为什么选择邮箱而非短信：**

| 对比项 | 短信验证码 | 邮箱验证码 |
|--------|-----------|-----------|
| 费用 | 0.045元/条 | **免费** |
| 每日限制 | 无限制（付费） | 免费邮箱 200-500 封/天 |
| 技术难度 | 中等（接入API） | **低**（标准库即可） |
| 到达率 | 较高 | 取决于邮箱服务商 |
| 适用场景 | 移动端、即时性要求高 | PC端、注册/找回密码 |

**邮箱验证码实现方案：**

```python
# backend/utils/email.py
import smtplib
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart
import random
import string

class EmailService:
    def __init__(self, smtp_server: str, smtp_port: int,
                 smtp_user: str, smtp_password: str):
        self.smtp_server = smtp_server      # 如：smtp.qq.com
        self.smtp_port = smtp_port          # 如：465 (SSL)
        self.smtp_user = smtp_user          # 发件邮箱
        self.smtp_password = smtp_password  # 邮箱授权码

    async def send_verification_code(self, to_email: str, code: str):
        """发送验证码邮件"""
        msg = MIMEMultipart('alternative')
        msg['Subject'] = 'ShareYourAi 验证码'
        msg['From'] = self.smtp_user
        msg['To'] = to_email

        # 纯文本版本
        text = f'您的验证码是：{code}，有效期5分钟。'
        # HTML版本
        html = f'''
        <div style="padding: 20px; background: #f5f5f5;">
            <h2>ShareYourAi 邮箱验证</h2>
            <p>您的验证码是：<strong style="font-size: 24px; color: #4F46E5;">{code}</strong></p>
            <p>验证码有效期 5 分钟，请勿泄露给他人。</p>
        </div>
        '''

        msg.attach(MIMEText(text, 'plain'))
        msg.attach(MIMEText(html, 'html'))

        # 发送邮件
        with smtplib.SMTP_SSL(self.smtp_server, self.smtp_port) as server:
            server.login(self.smtp_user, self.smtp_password)
            server.sendmail(self.smtp_user, to_email, msg.as_string())
```

**JavaScript 动效逻辑：**

```javascript
// 小人动效控制器
class CharacterAnimator {
    constructor() {
        this.characters = document.querySelectorAll('.character');
        this.charBodies = document.querySelectorAll('.char-body');
        this.eyeContainers = document.querySelectorAll('.eyes');
        this.eyePupils = document.querySelectorAll('.eye-pupil');
        this.bigEyes = document.querySelectorAll('.eye-big');
        this.smallEyes = document.querySelectorAll('.eye-small');
        this.mouth = document.querySelector('.mouth');

        this.isPeeking = false;
        this.isPasswordVisible = false;
        this.maxRotate = 15;
        this.minRotate = -15;

        this.init();
    }

    init() {
        this.initBlinkAnimation();
        this.bindMouseFollow();
        this.bindInputEvents();
        this.bindPasswordToggle();
    }

    // 1. 个性化眨眼动画（频率完全照搬原版）
    initBlinkAnimation() {
        // 紫色：6-8秒眨1次
        setInterval(() => {
            this.blinkMultipleTimes(
                document.querySelectorAll('.char-purple .eye-big'), 1
            );
        }, 6000 + Math.random() * 2000);

        // 黑色：4-5秒眨2次
        setInterval(() => {
            this.blinkMultipleTimes(
                document.querySelectorAll('.char-black .eye-big'), 2
            );
        }, 4000 + Math.random() * 1000);

        // 黄色：2-3秒眨3次
        setInterval(() => {
            this.blinkMultipleTimes(
                document.querySelectorAll('.char-yellow .eye-small'), 3
            );
        }, 2000 + Math.random() * 1000);

        // 橙色：8-10秒眨1次
        setInterval(() => {
            this.blinkMultipleTimes(
                document.querySelectorAll('.char-orange .eye-small'), 1
            );
        }, 8000 + Math.random() * 2000);
    }

    // 单次眨眼
    blinkEye(eyes, callback) {
        eyes.forEach(eye => {
            const originalHeight = this.isPeeking ?
                (eye.classList.contains('eye-big') ? '4.8px' : '3px') :
                (eye.classList.contains('eye-big') ? '3.6px' : '2.4px');

            eye.style.height = '0.6px';
            setTimeout(() => {
                eye.style.height = originalHeight;
                if (callback) callback();
            }, 100 + Math.random() * 50);
        });
    }

    // 多次眨眼
    blinkMultipleTimes(eyes, times) {
        let currentBlink = 0;
        const doBlink = () => {
            this.blinkEye(eyes, () => {
                currentBlink++;
                if (currentBlink < times) {
                    setTimeout(doBlink, 200 + Math.random() * 100);
                }
            });
        };
        doBlink();
    }

    // 2. 偷窥高频眨眼
    peekBlink() {
        let blinkCount = 0;
        const blinkInterval = setInterval(() => {
            this.bigEyes.forEach(eye => {
                eye.style.height = '0.6px';
                setTimeout(() => { eye.style.height = '4.8px'; }, 80);
            });
            document.querySelectorAll('.char-yellow .eye-small').forEach(eye => {
                eye.style.height = '0.6px';
                setTimeout(() => { eye.style.height = '3px'; }, 80);
            });
            document.querySelectorAll('.char-orange .eye-small').forEach(eye => {
                eye.style.height = '0.6px';
                setTimeout(() => { eye.style.height = '2.4px'; }, 80);
            });
            blinkCount++;
            if (blinkCount >= 3) clearInterval(blinkInterval);
        }, 200);
    }

    // 3. 切换偷窥状态
    togglePeeking(active) {
        this.isPeeking = active;
        this.characters.forEach(char => {
            char.classList.toggle('peeking', active);
        });
        if (active) {
            this.peekBlink();
        }
    }

    // 4. 鼠标跟随（角度范围完全照搬）
    updateCharacterSwing(mouseX, mouseY) {
        const container = document.querySelector('.characters');
        if (!container) return;

        const rect = container.getBoundingClientRect();
        const centerX = rect.left + rect.width / 2;
        const centerY = rect.top + rect.height / 2;

        const offsetX = (mouseX - centerX) / rect.width * 100;
        const offsetY = (mouseY - centerY) / rect.height * 100;

        // 身体摆动角度（±15°）
        const swingAngle = this.isPasswordVisible ?
            -8 : Math.max(this.minRotate, Math.min(this.maxRotate, offsetX / 5));

        // 更新小人身体
        this.charBodies.forEach((body, index) => {
            const swingCoeff = 1.5 - (index * 0.25);
            if (!this.isPeeking) {
                body.style.transform = `rotate(${swingAngle * swingCoeff}deg)`;
            }
        });

        // 更新眼睛位置
        this.eyeContainers.forEach((eyes, index) => {
            const eyeOffsetX = this.isPasswordVisible ? -1.5 : swingAngle / 1.8;
            const eyeOffsetY = this.isPasswordVisible ? 0 : offsetY / 6;

            const clampedX = Math.max(-0.6, Math.min(0.6, eyeOffsetX));
            const clampedY = Math.max(-0.6, Math.min(0.6, eyeOffsetY));

            if (!this.isPeeking) {
                eyes.querySelectorAll('.eye-small').forEach(eye => {
                    eye.style.transform = `translate(${eyeOffsetX}px, ${eyeOffsetY}px)`;
                });
                eyes.querySelectorAll('.eye-big').forEach(eye => {
                    eye.style.transform = `translate(${eyeOffsetX}px, ${eyeOffsetY}px)`;
                });
                this.eyePupils.forEach(pupil => {
                    pupil.style.transform =
                        `translate(-50%, -50%) translate(${clampedX}px, ${clampedY}px)`;
                });

                // 黄色小人嘴巴联动（90%幅度）
                if (this.mouth && index === 2) {
                    this.mouth.style.transform =
                        `translateX(-50%) translateX(${eyeOffsetX * 0.9}px) ` +
                        `translateY(${eyeOffsetY * 0.9}px) rotate(${eyeOffsetX * 0.09}deg)`;
                }
            }
        });
    }

    // 5. 绑定鼠标移动
    bindMouseFollow() {
        document.addEventListener('mousemove', (e) => {
            this.updateCharacterSwing(e.clientX, e.clientY);
        });
    }

    // 6. 绑定输入框事件（偷窥状态）
    bindInputEvents() {
        const inputs = document.querySelectorAll('input[type="text"], input[type="email"], input[type="password"]');
        inputs.forEach(input => {
            input.addEventListener('focus', () => this.togglePeeking(true));
            input.addEventListener('blur', () => this.togglePeeking(false));
        });
    }

    // 7. 绑定密码显隐切换
    bindPasswordToggle() {
        const toggleBtn = document.querySelector('.toggle-password');
        const passwordInput = document.querySelector('input[type="password"]');

        if (toggleBtn && passwordInput) {
            toggleBtn.addEventListener('click', () => {
                this.isPasswordVisible = !this.isPasswordVisible;
                passwordInput.type = this.isPasswordVisible ? 'text' : 'password';

                // 切换图标
                const icon = toggleBtn.querySelector('i');
                icon.classList.toggle('fa-eye');
                icon.classList.toggle('fa-eye-slash');
            });
        }
    }
}

// 初始化
document.addEventListener('DOMContentLoaded', () => {
    new CharacterAnimator();
});

```

```
│                                                             │

│       │                                                     │
│       ↓                                                     │
│  后端：                                                      │
│  - 生成 task_id                                             │
│  - 写入数据库（status: pending）                            │

│  - 触发派单                                                 │
│                                                             │
│  ─────────────────────────────────────                      │
│                                                             │
│  【阶段2：派单】                                             │
│                                                             │
│  派单引擎：                                                  │
│  - 根据配置的策略选择节点                                   │
│  - 更新节点状态 → busy                                      │
│  - WebSocket 推送任务                                       │
│                                                             │
│  ─────────────────────────────────────                      │
│                                                             │
│  【阶段3：节点执行】                                         │
│                                                             │
│  插件收到任务：                                              │
│  - 记录开始时间                                             │
│  - 执行页面操作（每步记录时间戳）                           │
│  - 等待视频生成                                             │
│  - 拦截 AI 请求响应数据                                     │
│  - 下载视频文件                                             │
│  - 上传到 COS                                               │
│  - 提交完整链路数据                                         │
│                                                             │
│  ─────────────────────────────────────                      │
│                                                             │
│  【阶段4：后端校验】                                         │
│                                                             │
│  收到 success 响应：                                        │
│       │                                                     │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 1: 时间合理性                                   │   │
│  │ - 耗时 >= 最小值                                    │   │
│  │ - 耗时 <= 最大值                                    │   │
│  │ - 各步骤时间顺序正确                                │   │
│  └─────────────────────────────────────────────────────┘   │
│       │ 通过                                                 │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 2: 文件有效性                                   │   │
│  │ - 从 COS 获取文件信息                               │   │
│  │ - 文件大小在允许范围内                              │   │
│  │ - 文件头是有效视频/图片格式                          │   │
│  └─────────────────────────────────────────────────────┘   │
│       │ 通过                                                 │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 3: 链路完整性                                   │   │
│  │ - 所有必需步骤都有记录                              │   │
│  │ - video_detected_time 存在（视频确实出现在页面）     │   │
│  │ - 拦截到 AI 请求响应数据                            │   │
│  └─────────────────────────────────────────────────────┘   │
│       │ 通过                                                 │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 校验 4: 频率限制                                     │   │
│  │ - 节点本小时任务数 <= 限制                          │   │
│  │ - 用户本小时任务数 <= 限制                          │   │
│  └─────────────────────────────────────────────────────┘   │
│       │                                                     │
│       ├── 全部通过 → 进入审核流程                          │
│       │                                                     │
│       └── 任一失败 → 标记异常，记录风控日志                │
│                                                             │
│  ─────────────────────────────────────                      │
│                                                             │
│  【阶段5：收益审核】                                         │
│                                                             │
│  自动审核规则：                                              │
│  - 用户信用良好 + 校验全部通过 → 自动通过                   │
│  - 新用户 + 首次任务 → 人工审核                             │
│  - 校验有警告项 → 人工审核                                  │
│  - 高频任务 → 人工审核                                      │
│                                                             │
│  审核结果：                                                  │
│  - 通过 → earning_status: settled（冻结期开始）            │
│  - 拒绝 → earning_status: cancelled                        │
│                                                             │
│  ─────────────────────────────────────                      │
│                                                             │
│  【阶段6：冻结期与提现】                                     │
│                                                             │
│  settled 状态：                                              │
│  - 收益进入 frozen_settled                                  │
│  - 冻结期 T+3 天                                            │
│                                                             │
│  冻结期结束：                                                │
│  - frozen_settled → withdrawable                           │
│  - earning_status: withdrawable                             │
│                                                             │
│  用户可发起提现：                                            │
│  - 提现金额从 withdrawable 扣除                             │
│  - 进入 frozen_withdrawing                                  │
│  - 管理员打款后完成                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
